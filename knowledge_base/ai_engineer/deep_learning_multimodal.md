## Q1: 深度学习中的优化算法对比

**规范答案：**

**梯度下降变体：**
| 算法 | 特点 | 优缺点 |
|------|------|--------|
| SGD | 每个样本更新一次 | 简单但震荡大 |
| Mini-batch SGD | 小批量更新 | 平衡效率和稳定性 |
| Momentum | 加入动量项，加速收敛 | 减少震荡 |
| AdaGrad | 自适应学习率（累积梯度平方） | 适合稀疏特征，后期学习率趋零 |
| RMSProp | 指数加权平均修正 AdaGrad | 适合非平稳目标 |
| **Adam** | Momentum + RMSProp | 最常用，默认选择 |
| AdamW | Adam + 权重衰减（解耦正则化） | 大模型训练标准 |

**Adam 算法核心：**
- 一阶矩估计（均值） + 二阶矩估计（方差）
- 偏差修正：因初始化为 0 的偏差进行修正
- 超参数：lr=1e-3, β1=0.9, β2=0.999, ε=1e-8

**学习率调度：**
- Warmup：初始阶段线性增大学习率
- Cosine Annealing：余弦退火，周期性降低学习率
- StepLR、ReduceLROnPlateau

**得分点：**
- [ ] 至少了解 SGD、Adam 的原理
- [ ] 知道 Adam 是 Momentum + RMSProp 结合
- [ ] 理解学习率调度的作用
- [ ] 知道为什么大模型训练用 AdamW

**加分项：**
- 分析 Adam 的收敛性问题和 AdamW 的改进
- 提到 LAMB / LARS 大批量训练优化器
- 讨论学习率 Warmup 的必要性

**追问方向：**
- 为什么大模型需要 Warmup？
- 梯度消失和梯度爆炸如何解决？
- 批量大小对训练有什么影响？

## Q2: CNN 经典架构与核心概念

**规范答案：**

**核心组件：**
1. **卷积层（Conv）**：局部感受野 + 权值共享，提取空间特征
2. **池化层（Pooling）**：下采样，减少参数，增加平移不变性
3. **全连接层（FC）**：最终分类/回归
4. **Batch Normalization**：稳定训练，加速收敛
5. **激活函数**：ReLU（最常用），解决梯度消失

**经典架构发展：**
| 模型 | 年份 | 核心创新 |
|------|------|---------|
| LeNet | 1998 | 开创性的 CNN |
| AlexNet | 2012 | ReLU、Dropout、GPU 训练 |
| VGGNet | 2014 | 小卷积核堆叠（3x3）|
| GoogLeNet | 2014 | Inception 模块（多尺度） |
| ResNet | 2015 | 残差连接，训练超深网络 |
| EfficientNet | 2019 | 复合缩放（宽度×深度×分辨率） |

**ResNet 残差连接：**
- 核心思想：学习残差 F(x) = H(x) - x，而非直接学习 H(x)
- 恒等映射捷径使梯度直接传播，解决退化问题
- 使训练 100+ 层网络成为可能

**得分点：**
- [ ] 理解卷积操作的本质和优势
- [ ] 知道 ResNet 残差连接的原理
- [ ] 了解 CNN 架构的发展脉络
- [ ] 知道 BN 层的作用

**加分项：**
- 分析感受野的计算方法
- 对比 ViT 和 CNN 的优劣
- 提到深度可分离卷积（MobileNet）

**追问方向：**
- 1x1 卷积有什么用？
- CNN 和 Transformer 在视觉任务上的对比？
- 转置卷积（反卷积）的原理和用途？

## Q3: 多模态大模型

**规范答案：**

**多模态模型的定义：**
能处理和理解多种数据模态（文本、图像、音频、视频）的 AI 模型。

**核心架构模式：**
1. **双塔结构（CLIP 系列）**：
   - 文本编码器 + 图像编码器，对比学习对齐表示空间
   - 应用：图文检索、零样本分类
2. **编码器-解码器结构（LLaVA / GPT-4V）**：
   - 视觉编码器（ViT）提取图像特征 → 投影层 → LLM 理解和生成
   - 应用：图像问答、图像描述
3. **统一序列结构（Gemini）**：
   - 所有模态统一为 token 序列，端到端训练

**代表模型：**
| 模型 | 类型 | 模态 | 特点 |
|------|------|------|------|
| CLIP | 对比学习 | 图+文 | 零样本分类能力强 |
| GPT-4V/4o | 生成式 | 图+文+音 | 最强多模态理解 |
| LLaVA | 开源 | 图+文 | 高效微调方案 |
| Whisper | 语音 | 音+文 | 多语言语音识别 |
| Stable Diffusion | 生成式 | 文→图 | 文本生成图像 |

## Q4: Diffusion Model 原理

**规范答案：**

**核心思想：**
通过逐步添加噪声（前向过程）和学习去噪（反向过程）来生成数据。

**前向过程（加噪）：**
- 在原始数据上逐步添加高斯噪声
- 经过 T 步后变为纯高斯噪声
- 每步噪声由调度器（scheduler）控制

**反向过程（去噪）：**
- 训练一个神经网络（通常是 U-Net），预测每步的噪声
- 从纯噪声开始，逐步去噪恢复原始数据
- 每步预测的是噪声（ε-prediction），而非直接预测干净图

**条件生成（Classifier-Free Guidance）：**
- 同时训练有条件和无条件模型
- 推理时插值：输出 = 无条件 + guidance_scale × (有条件 - 无条件)
- guidance_scale 越大，生成越贴合条件但多样性降低

**Stable Diffusion 架构：**
1. 文本编码器（CLIP Text Encoder）→ 文本嵌入
2. U-Net：在潜空间（Latent Space）中去噪
3. VAE Decoder：从潜空间解码到像素空间
4. 在潜空间操作大幅降低计算成本

**得分点：**
- [ ] 理解前向-反向过程的基本思想
- [ ] 知道模型学习的是预测噪声
- [ ] 了解 Stable Diffusion 在潜空间操作的设计
- [ ] 知道 Classifier-Free Guidance 的作用

**加分项：**
- 分析 DDPM 和 DDIM 采样策略的区别
- 讨论 ControlNet、LoRA 等条件控制方法
- 提到 Flow Matching 和 Rectified Flow

**追问方向：**
- 为什么在潜空间操作比像素空间更高效？
- ControlNet 是如何实现条件控制的？
- Diffusion Model 和 GAN 的对比？

## Q5: 语音识别与 TTS 技术

**规范答案：**

**ASR（自动语音识别）：**

| 方法 | 代表模型 | 特点 |
|------|---------|------|
| 传统流水线 | Kaldi | 声学模型+语言模型+解码器 |
| 端到端（CTC） | DeepSpeech | CTC 损失函数 |
| 端到端（Seq2Seq） | Listen-Attend-Spell | 注意力机制 |
| 大规模预训练 | **Whisper** | 多语言、多任务、鲁棒性强 |

**Whisper 模型：**
- OpenAI 开源，68 万小时多语言数据训练
- Encoder-Decoder 架构（Transformer）
- 支持：语音识别、翻译、语言检测、时间戳
- 多种模型大小：tiny → large-v3

**TTS（文本转语音）：**
- 传统：文本分析 → 声学模型 → 声码器
- 现代端到端：VITS、Bark、ChatTTS
- 零样本语音克隆：只需几秒参考音频

**得分点：**
- [ ] 了解 ASR 的基本流程
- [ ] 知道 Whisper 的核心特点

## Q6: AI 工程开发实践

**规范答案：**

**LLM 应用开发技术栈：**
| 层级 | 技术选项 |
|------|---------|
| LLM API | OpenAI API / Claude API / 本地部署 |
| 开发框架 | LangChain / LlamaIndex |
| 向量数据库 | Milvus / ChromaDB / Pinecone |
| Agent 框架 | LangGraph / CrewAI |
| 前端 | Streamlit / Gradio / Next.js |
| 部署 | vLLM / TensorRT-LLM / Ollama |
| 评估 | RAGAS / LangSmith / OpenAI Evals |

**开发最佳实践：**
1. **评估驱动开发**：先建立评估体系（自动 + 人工），再迭代优化
2. **成本控制**：选择合适的模型（不一定最大最好），缓存重复请求
3. **安全防护**：输入过滤、输出审查、Prompt 注入防护
4. **可观测性**：日志记录所有 LLM 调用的输入输出和延迟
5. **错误处理**：重试机制、降级方案、速率限制

**模型选型策略：**
- 简单任务（分类、提取）→ 小模型或微调模型
- 复杂推理 → 大模型（GPT-4、Claude）
- 隐私敏感 → 本地部署（Llama、Qwen）
- 实时要求 → 小模型 + 量化
