# 计算机基础 - 数据结构与算法面试知识库

## 1. 核心数据结构

### 数组 vs 链表
| 维度 | 数组 | 链表 |
|------|------|------|
| 内存 | 连续 | 离散 |
| 随机访问 | O(1) | O(n) |
| 插入/删除 | O(n) | O(1)（已定位） |
| 缓存友好 | 高（局部性原理） | 低 |

### 栈与队列
- **栈 (Stack)**：LIFO，应用：表达式求值、括号匹配、函数调用栈、浏览器前进后退
- **队列 (Queue)**：FIFO，应用：BFS、任务调度、消息队列
- **单调栈**：找下一个更大/更小元素，O(n)
- **优先队列 (Heap)**：取最值 O(1)，插入/删除 O(logN)

### 哈希表 (HashMap)
- 哈希冲突解决：拉链法（链表/红黑树）、开放寻址法（线性探测/二次探测）
- 扩容：负载因子 > 0.75 时 2 倍扩容，rehash
- 时间复杂度：平均 O(1)，最坏 O(n)

### 树
- **二叉搜索树 (BST)**：左 < 根 < 右，查找/插入 O(logN)~O(N)
- **平衡树 (AVL/红黑树)**：保证 O(logN)
- **B+ 树**：多路平衡搜索树，数据库索引专用
- **Trie (前缀树)**：字符串高效检索

### 图
- **存储**：邻接矩阵（稠密图）/ 邻接表（稀疏图）
- **遍历**：BFS（广度优先，队列）/ DFS（深度优先，栈/递归）
- **最短路径**：Dijkstra（非负权）/ Bellman-Ford（有负权）/ Floyd（多源）
- **最小生成树**：Prim / Kruskal
- **拓扑排序**：DAG 中的任务依赖排序

### 标准答案要点（5星回答）
1. 各数据结构的时间复杂度对比
2. HashMap 的扩容机制和冲突解决
3. 红黑树的 5 个性质和旋转操作
4. B+ 树为什么适合数据库索引
5. 图算法在实际工程中的应用

## 2. 核心算法思想

### 排序算法
| 算法 | 平均 | 最坏 | 空间 | 稳定 |
|------|------|------|------|------|
| 冒泡排序 | O(n²) | O(n²) | O(1) | ✓ |
| 选择排序 | O(n²) | O(n²) | O(1) | ✗ |
| 插入排序 | O(n²) | O(n²) | O(1) | ✓ |
| 快速排序 | O(nlogn) | O(n²) | O(logn) | ✗ |
| 归并排序 | O(nlogn) | O(nlogn) | O(n) | ✓ |
| 堆排序 | O(nlogn) | O(nlogn) | O(1) | ✗ |

### 算法范式
1. **双指针**：有序数组两数之和、链表快慢指针
2. **滑动窗口**：最长/最短子串/子数组
3. **二分查找**：有序序列查找、查找边界
4. **BFS/DFS**：树/图遍历、路径搜索
5. **动态规划**：最优子结构 + 重叠子问题
6. **贪心算法**：局部最优 → 全局最优
7. **回溯法**：排列组合、N 皇后
8. **分治法**：归并排序、快排

### 动态规划框架
```
dp[i] = 最优值
base case: dp[0] = ...
状态转移: dp[i] = f(dp[i-1], dp[i-2], ...)
```

**经典题型：**
- 斐波那契 / 爬楼梯
- 背包问题（0-1 背包、完全背包）
- 最长公共子序列 (LCS)
- 最长递增子序列 (LIS)
- 编辑距离
- 股票买卖

### 标准答案要点（5星回答）
1. 各排序算法的复杂度和稳定性
2. 快排的 partition 过程和优化
3. 动态规划的解题框架
4. 至少 5 种算法范式及典型应用

## 3. 操作系统基础

### 进程 vs 线程 vs 协程
| 维度 | 进程 | 线程 | 协程 |
|------|------|------|------|
| 资源 | 独立地址空间 | 共享进程地址空间 | 共享线程栈 |
| 切换开销 | 大（内核态） | 中（内核态） | 小（用户态） |
| 通信 | IPC（管道/信号/共享内存） | 共享内存 + 同步原语 | channel/yield |
| 创建开销 | 大 | 中 | 小（KB级栈） |

### 内存管理
- **虚拟内存**：每个进程独立的虚拟地址空间 → 页表映射 → 物理内存
- **分页 (Paging)**：固定大小页面（4KB），页表映射
- **缺页中断**：访问不在物理内存中的页 → 从磁盘加载
- **页面置换算法**：LRU（最近最少使用）/ FIFO（先进先出）/ Clock

### 进程调度
- **FCFS**：先来先服务
- **SJF**：短作业优先
- **RR**：时间片轮转
- **多级反馈队列**：Linux CFS（完全公平调度器）

1. 进程/线程/协程的区别及适用场景
2. 虚拟内存和页表的工作方式

4. Linux 常用命令（ps/top/netstat/awk/sed）

## 4. 计算机网络

### TCP 三次握手
```
Client           Server
  |--- SYN --->    |   1. 客户端发送 SYN=1, seq=x
  |<-- SYN+ACK --- |   2. 服务端回复 SYN=1, ACK=1, seq=y, ack=x+1
  |--- ACK --->    |   3. 客户端确认 ACK=1, seq=x+1, ack=y+1
```
**为什么不是两次？** 防止旧的重复连接请求到达服务端，导致资源浪费

### TCP 四次挥手
```
Client           Server
  |--- FIN --->    |   1. 客户端请求关闭
  |<-- ACK ---     |   2. 服务端确认（此时服务端可能还有数据要发）
  |<-- FIN ---     |   3. 服务端也请求关闭
  |--- ACK --->    |   4. 客户端确认（进入 TIME_WAIT 2MSL）
```
**为什么需要 TIME_WAIT？** 1) 确保最后一个 ACK 到达 2) 防止旧连接的延迟包影响新连接

### TCP 拥塞控制
- **慢启动**：cwnd 从 1 开始，每 RTT 翻倍
- **拥塞避免**：超过 ssthresh 后线性增长
- **快速重传**：收到 3 个重复 ACK 立即重传
- **快速恢复**：ssthresh = cwnd/2，cwnd = ssthresh + 3

### TCP vs UDP
| 维度 | TCP | UDP |
|------|-----|-----|
| 连接 | 面向连接 | 无连接 |
| 可靠性 | 保证有序可靠 | 不保证 |
| 头部开销 | 20 字节 | 8 字节 |
| 应用 | HTTP/FTP/SSH | DNS/视频流/游戏/QUIC |

2. TCP 可靠传输机制（重传、流量控制、拥塞控制）
3. HTTP 各版本的演进
4. OSI 七层模型或 TCP/IP 四层模型
5. 有抓包分析（Wireshark/tcpdump）的经验
