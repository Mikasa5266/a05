## 1. Vue 3 响应式原理

### 第一层：概念确认
**问题**: "Vue 3 的响应式系统是基于什么实现的？和 Vue 2 有什么区别？"
**期望要点**: Vue 3 用 Proxy 代替 Vue 2 的 Object.defineProperty；支持动态属性、数组、Map/Set
**评分标准**: 说清 Proxy 和 defineProperty 的差异得 3 分

### 第二层：原理深挖
**触发条件**: 候选人了解响应式基础
**问题**: "Vue 3 的 ref 和 reactive 有什么区别？为什么有时候需要用 .value？"
**期望要点**: ref 用于基本类型（包装为 {value: x}）；reactive 用于对象；ref 在模板中自动解包
**评分标准**: 解释清楚 ref 包装原因得 4 分，说清收集依赖的 track/trigger 机制得 5 分

### 第三层：实战应用
**触发条件**: 候选人掌握响应式原理
**问题**: "在大型项目中，如何避免响应式数据导致的性能问题？你用过 shallowRef 或 markRaw 吗？"
**期望要点**: shallowRef 避免深层代理；markRaw 跳过代理；computed 缓存；watchEffect 清理
**评分标准**: 有实际性能优化案例得 5 分

## 2. Virtual DOM 与 Diff 算法

### 第一层：概念确认
**问题**: "什么是 Virtual DOM？为什么需要它？"
**期望要点**: 用 JS 对象描述 DOM 树；减少直接 DOM 操作的性能开销
**评分标准**: 说清 VDOM 定义和目的得 3 分

### 第二层：原理深挖
**触发条件**: 候选人了解 VDOM 概念
**问题**: "Vue 3 的 Diff 算法和 Vue 2 有什么不同？最长递增子序列在其中扮演什么角色？"
**期望要点**: Vue 3 使用静态标记 + 最长递增子序列优化移动操作；PatchFlag 跳过静态节点
**评分标准**: 说清 PatchFlag 和 LIS 优化得 5 分

### 第三层：实战应用
**触发条件**: 候选人掌握 Diff 原理
**问题**: "在实际开发中，你是怎么写出对 Diff 性能更友好的组件的？列表渲染有哪些注意事项？"
**期望要点**: 保持稳定 key、避免 index 作 key、v-once/v-memo 静态优化、组件拆分
**评分标准**: 有丰富的优化实践得 5 分

## 3. 浏览器事件循环

### 第一层：概念确认
**问题**: "JavaScript 的事件循环是什么？宏任务和微任务有什么区别？"
**期望要点**: 单线程+事件队列；宏任务（setTimeout, setInterval）、微任务（Promise.then, MutationObserver）
**评分标准**: 说清宏/微任务分类得 3 分

### 第二层：原理深挖
**触发条件**: 候选人了解事件循环基础
**问题**: "请说明这段代码的输出顺序：`console.log(1); setTimeout(()=>console.log(2),0); Promise.resolve().then(()=>console.log(3)); console.log(4);`"
**期望要点**: 输出 1, 4, 3, 2；同步代码先执行，微任务优先于宏任务
**评分标准**: 输出正确且能解释原因得 4 分

### 第三层：实战应用
**触发条件**: 候选人能正确分析执行顺序
**问题**: "Vue 的 nextTick 是基于什么实现的？为什么 DOM 更新是异步的？"
**期望要点**: nextTick 优先用 Promise.then，降级 MutationObserver → setImmediate → setTimeout；异步批量更新提升性能
**评分标准**: 说清 nextTick 实现原理和源码细节得 5 分

## 5. CSS 布局

### 第一层：概念确认
**问题**: "Flexbox 和 Grid 布局的核心区别是什么？分别适合什么场景？"
**期望要点**: Flex 是一维布局；Grid 是二维布局；Flex 适合行/列排布，Grid 适合复杂网格
**评分标准**: 说清一维 vs 二维得 3 分

### 第二层：原理深挖
**触发条件**: 候选人了解区别
**问题**: "Flex 的 `flex: 1` 等价于什么？flex-grow、flex-shrink、flex-basis 分别什么含义？"
**期望要点**: flex: 1 = flex-grow:1 flex-shrink:1 flex-basis:0%；grow 分配剩余空间/shrink 收缩比/basis 初始大小
**评分标准**: 三个属性全部解释正确得 4 分

### 第三层：实战应用
**触发条件**: 候选人掌握 Flex 属性
**问题**: "如何实现一个顶部固定、左侧导航固定、右侧内容可滚动的经典后台布局？"
**期望要点**: 使用 Flex/Grid 布局，overflow-y: auto 处理滚动，vh 单位，sticky 定位
**评分标准**: 能给出多种方案并分析优劣得 5 分
