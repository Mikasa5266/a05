# 追问逻辑链 - Java 后端

> 每个知识点设计 3 层递进追问：概念层 → 原理层 → 实战层，用于面试官 AI 层层深挖。

---

## 1. JVM 内存模型

### 第一层：概念确认
**问题**: "请描述 JVM 的内存区域划分。"
**期望要点**: 堆、栈、方法区/元空间、程序计数器、本地方法栈
**评分标准**: 能列出 5 个区域得 3 分，说清每个区域存什么得 4 分

### 第二层：原理深挖
**触发条件**: 候选人正确列出了内存分区
**问题**: "堆内存的分代模型是怎样的？新生代为什么要分 Eden 和两个 Survivor？"
**期望要点**: 新生代（Eden:S0:S1=8:1:1）、老年代；复制算法只需要 10% 空间做担保
**评分标准**: 答出分代结构得 3 分，解释复制算法为什么需要两个 Survivor 得 5 分

### 第三层：实战应用
**触发条件**: 候选人能解释分代模型
**问题**: "你在实际项目中遇到过 GC 性能问题吗？具体是怎么排查和优化的？"
**期望要点**: jstat/jmap/MAT 等工具、具体调优参数、优化前后对比数据
**评分标准**: 有真实案例和量化数据得 5 分

## 2. 并发编程 - synchronized

### 第一层：概念确认
**问题**: "synchronized 关键字的作用是什么？可以修饰哪些对象？"
**期望要点**: 互斥锁、可修饰方法/代码块，对象锁 vs 类锁
**评分标准**: 说清互斥语义和使用方式得 3 分

### 第二层：原理深挖
**触发条件**: 候选人了解基本使用
**问题**: "JDK 1.6 后 synchronized 做了哪些优化？锁升级过程是怎样的？"
**期望要点**: 偏向锁→轻量级锁→重量级锁；Mark Word 中的锁标志位变化
**评分标准**: 完整描述三级升级链路得 4 分，说清 Mark Word 结构得 5 分

### 第三层：实战应用
**触发条件**: 候选人能说清锁升级
**问题**: "在什么场景下 synchronized 性能不如 ReentrantLock？为什么？"
**期望要点**: 需要尝试获取锁(tryLock)、超时、中断响应、公平锁、多条件变量的场景
**评分标准**: 能从功能和性能两个维度对比并给出场景建议得 5 分

## 3. Spring - IoC 容器

### 第一层：概念确认
**问题**: "什么是 IoC（控制反转）？它和 DI（依赖注入）是什么关系？"
**期望要点**: IoC 是把对象创建权交给容器；DI 是 IoC 的实现方式之一
**评分标准**: 区分 IoC 和 DI 得 3 分

### 第二层：原理深挖
**触发条件**: 候选人理解 IoC 概念
**问题**: "Spring Bean 的完整生命周期是怎样的？从创建到销毁经历了哪些步骤？"
**期望要点**: 实例化→属性注入→Aware 回调→BeanPostProcessor 前置→InitializingBean→自定义 init→BPP 后置→就绪→DisposableBean→自定义 destroy
**评分标准**: 答出 6 个以上步骤得 4 分，解释 BeanPostProcessor 的切入时机得 5 分

### 第三层：实战应用
**触发条件**: 候选人了解生命周期
**问题**: "在 Spring 中如何解决循环依赖问题？三级缓存分别存放什么？"
**期望要点**: singletonObjects/earlySingletonObjects/singletonFactories；通过提前暴露 ObjectFactory 解决
**评分标准**: 说清三级缓存内容和解决流程得 5 分

## 4. MySQL - 索引优化

### 第一层：概念确认
**问题**: "MySQL 的索引底层使用什么数据结构？为什么选择 B+ 树而不是 B 树或 Hash？"
**期望要点**: B+ 树；叶子节点串联便于范围查询；非叶节点只存键可以更矮
**评分标准**: 答出 B+ 树并给出至少 2 个理由得 3 分

### 第二层：原理深挖
**触发条件**: 候选人了解 B+ 树
**问题**: "什么是索引的最左前缀匹配原则？哪些情况下索引会失效？"
**期望要点**: 联合索引按最左列匹配；LIKE '%xx'、函数包裹、隐式类型转换、OR 混用导致失效
**评分标准**: 说出最左前缀原则得 3 分，列出 4 个以上失效场景得 5 分

### 第三层：实战应用
**触发条件**: 候选人掌握索引原理
**问题**: "请用 EXPLAIN 分析一条慢 SQL 并说明优化思路。你通常关注哪些字段？"
**期望要点**: type（ALL→ref→const）、key、rows、Extra（Using filesort/temporary）
**评分标准**: 能解读 EXPLAIN 关键字段并给出具体优化手段得 5 分

## 5. Redis - 缓存策略

### 第一层：概念确认
**问题**: "什么是缓存穿透、缓存击穿和缓存雪崩？它们有什么区别？"
**期望要点**: 穿透=查不存在的数据；击穿=热点key过期；雪崩=大量key同时过期
**评分标准**: 准确区分三者得 3 分

### 第二层：原理深挖
**触发条件**: 候选人能区分三个概念
**问题**: "针对这三个问题，分别有哪些解决方案？各自的优缺点是什么？"
**期望要点**: 穿透→布隆过滤器/缓存空值；击穿→互斥锁/逻辑过期；雪崩→随机过期时间/多级缓存
**评分标准**: 每类给出 2 个以上方案并分析优缺点得 5 分

### 第三层：实战应用
**触发条件**: 候选人掌握解决方案
**问题**: "你在项目中是如何设计缓存与数据库的一致性方案的？用的什么更新策略？"
**期望要点**: Cache Aside/Write Through/Write Behind；延迟双删；binlog 订阅
**评分标准**: 有实际方案选型经验并能分析 trade-off 得 5 分

## 6. 项目经验 - 系统设计

### 第三层：实战应用

**问题**: "如果重新做这个项目，你会改进哪些地方？为什么？"
**期望要点**: 技术选型反思、架构改进、流程优化，体现成长性思维
**评分标准**: 有具体改进点和理由，展现技术判断力得 5 分
