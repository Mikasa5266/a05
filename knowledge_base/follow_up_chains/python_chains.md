## 1. GIL 全局解释器锁

### 第一层：概念确认
**问题**: "Python 的 GIL 是什么？它对多线程有什么影响？"
**期望要点**: GIL 是 CPython 的全局互斥锁，同一时刻只有一个线程执行 Python 字节码
**评分标准**: 说清 GIL 定义和影响得 3 分

### 第二层：原理深挖
**触发条件**: 候选人了解 GIL 基本概念
**问题**: "GIL 是在什么粒度上释放的？CPU 密集型和 IO 密集型任务分别应该用什么并发方案？"
**期望要点**: 每执行一定数量字节码或 IO 操作时释放；CPU 密集→multiprocessing；IO 密集→threading/asyncio
**评分标准**: 区分场景并给出方案得 4 分，提到 asyncio 协程得 5 分

### 第三层：实战应用
**触发条件**: 候选人掌握并发方案
**问题**: "你在实际项目中是如何绕过 GIL 限制的？用的什么方案？效果如何？"
**期望要点**: 实际案例（如 Celery 任务队列、C 扩展、ProcessPoolExecutor），有性能数据
**评分标准**: 有真实场景和量化数据得 5 分

## 2. 装饰器与元类

### 第一层：概念确认
**问题**: "Python 装饰器的本质是什么？请举一个实际应用场景。"
**期望要点**: 装饰器本质是高阶函数，接收函数返回函数；应用如日志、缓存、权限校验
**评分标准**: 说清本质并给出例子得 3 分

### 第二层：原理深挖
**触发条件**: 候选人理解装饰器基本用法
**问题**: "如何实现一个带参数的装饰器？`functools.wraps` 的作用是什么？"
**期望要点**: 三层嵌套函数；wraps 保留原函数的 __name__、__doc__ 等元信息
**评分标准**: 能写出三层结构并解释 wraps 得 4 分

### 第三层：实战应用
**触发条件**: 候选人掌握装饰器进阶
**问题**: "什么是元类？`__new__` 和 `__init__` 的区别是什么？ORM 框架如何利用元类？"
**期望要点**: 元类是创建类的类；__new__ 创建实例，__init__ 初始化；Django ORM 用元类收集字段定义
**评分标准**: 说清元类原理并举出框架应用得 5 分

## 4. 异步编程 asyncio

### 第一层：概念确认
**问题**: "Python 的 asyncio 是什么？async/await 关键字分别表示什么？"
**期望要点**: asyncio 是 Python 的事件循环库；async 定义协程函数，await 挂起等待异步操作
**评分标准**: 说清事件循环概念得 3 分

### 第二层：原理深挖
**触发条件**: 候选人了解基本概念
**问题**: "协程和线程的区别是什么？asyncio 的事件循环是如何调度协程的？"
**期望要点**: 协程是用户态调度/更轻量/协作式；事件循环通过 epoll/select 监听 IO 事件
**评分标准**: 从调度方式、内存开销、切换成本三方面对比得 5 分

### 第三层：实战应用
**触发条件**: 候选人理解协程原理
**问题**: "在 asyncio 中如何正确处理阻塞 IO？如何避免协程中的常见陷阱？"
**期望要点**: run_in_executor 包装阻塞调用；常见陷阱包括忘记 await、在协程中调用同步阻塞函数
**评分标准**: 有实际排错经验得 5 分
