## Q1: 动态规划的核心思想和解题框架

**规范答案：**

**核心思想：**
将原问题分解为相互重叠的子问题，通过存储子问题的解（记忆化），避免重复计算。

**与其他算法的区分：**
- **分治**：子问题不重叠（如归并排序）
- **贪心**：每步做局部最优选择，不保证全局最优
- **DP**：子问题重叠 + 最优子结构，保证全局最优

**解题框架（五步法）：**
1. **定义状态**：明确 `dp[i]` 或 `dp[i][j]` 的含义
2. **状态转移方程**：找出 `dp[i]` 与前一状态的关系
3. **初始条件**：确定 base case
4. **遍历顺序**：确保计算 `dp[i]` 时所依赖的状态已计算
5. **返回结果**：确定最终答案的位置

**经典问题分类：**
| 类型 | 经典题目 | 状态定义 |
|------|---------|---------|
| 线性 DP | 最长递增子序列（LIS） | `dp[i]` = 以 i 结尾的 LIS 长度 |
| 背包 DP | 0-1 背包、完全背包 | `dp[i][j]` = 前 i 个物品容量 j 的最大价值 |
| 区间 DP | 矩阵链乘法、戳气球 | `dp[i][j]` = 区间 [i,j] 的最优解 |
| 树形 DP | 打家劫舍 III | `dp[node]` = 以 node 为根的子树最优解 |
| 状态压缩 DP | TSP 问题 | 用位掩码表示状态 |

**得分点：**
- [ ] 清楚 DP 的两个核心性质（最优子结构 + 重叠子问题）
- [ ] 能描述完整的解题框架
- [ ] 能区分 DP、贪心、分治
- [ ] 至少能用一个经典问题举例说明

**加分项：**
- 能分析空间优化（滚动数组 / 一维优化）
- 能手写 LIS 的 O(n log n) 解法（二分 + 贪心）
- 讨论记忆化搜索 vs 自底向上 DP 的优劣

**追问方向：**
- 0-1 背包问题如何优化为一维空间？
- 最长公共子序列（LCS）的状态转移方程是什么？
- 如何判断一个问题是否能用 DP 解决？

## Q2: 常见图算法

**规范答案：**

**图的表示：**
- 邻接矩阵：空间 O(V²)，查边 O(1)，适合稠密图
- 邻接表：空间 O(V+E)，适合稀疏图

**图遍历：**
- **BFS（广度优先）**：队列实现，适合最短路径（无权图）、层序遍历
- **DFS（深度优先）**：递归/栈实现，适合连通分量、拓扑排序、环检测

**最短路径算法：**
| 算法 | 适用场景 | 时间复杂度 |
|------|---------|-----------|
| Dijkstra | 单源最短路（非负权） | O((V+E) log V) |
| Bellman-Ford | 单源最短路（可负权） | O(VE) |
| Floyd-Warshall | 全源最短路 | O(V³) |
| A* | 启发式搜索 | 取决于启发函数 |

**其他重要算法：**
- 拓扑排序：有向无环图（DAG）的线性排序，用于任务调度
- 最小生成树：Kruskal（按边排序）、Prim（按点扩展）
- 并查集：用于连通性判断、最小生成树

**得分点：**
- [ ] 知道两种图的表示方式及适用场景
- [ ] 理解 BFS 和 DFS 的区别和适用场景
- [ ] 至少掌握一种最短路径算法
- [ ] 知道拓扑排序的概念和应用

**加分项：**
- 能分析 Dijkstra 和 Bellman-Ford 的时间复杂度
- 提到 A* 算法和启发式函数
- 能应用图算法解决实际问题（如课程安排、网络路由）

**追问方向：**
- 如何检测图中是否有环？
- Dijkstra 为什么不能处理负权边？
- 并查集的路径压缩和按秩合并优化？

## Q3: 二分查找及其变体

**规范答案：**

**基础二分查找：**
在有序数组中查找目标值，时间复杂度 O(log n)。

**关键要点：**
- 循环条件：`left <= right`（闭区间）或 `left < right`（左闭右开）
- mid 计算：`left + (right - left) / 2`（防止溢出）
- 边界更新：根据题意更新 left 或 right

**常见变体：**
1. **查找第一个等于 target 的位置**（左边界）
2. **查找最后一个等于 target 的位置**（右边界）
3. **查找第一个大于等于 target 的位置**（lower_bound）
4. **查找最后一个小于等于 target 的位置**

**二分答案思想：**
当问题的答案具有单调性时，可以对答案进行二分：
- 例如：分割数组的最大值最小化、运输船的最小载重

**得分点：**
- [ ] 能正确写出基础二分查找（无 bug）
- [ ] 理解循环不变量和边界条件
- [ ] 知道至少两种变体
- [ ] 理解二分答案的思想

**加分项：**
- 能在旋转排序数组中使用二分查找
- 分析二分查找的模板化写法
- 提到实数域上的二分（精度控制）

**追问方向：**
- 在旋转排序数组中查找目标值如何做？
- 如何用二分查找求平方根？
- 二分查找在实际工程中有哪些应用？

## Q4: 常见算法思想对比（贪心、回溯、分治）

**规范答案：**

**贪心算法：**
- 每步选择当前最优，期望获得全局最优
- 适用条件：贪心选择性质 + 最优子结构
- 经典题目：活动选择、跳跃游戏、分配饼干、加油站问题
- 难点：证明贪心策略的正确性

**回溯算法：**
- 系统地搜索所有解，不满足条件时「剪枝」回退
- 本质是 DFS + 剪枝
- 经典题目：N 皇后、全排列、组合总和、数独
- 模板：做选择 → 递归 → 撤销选择

**分治算法：**
- 将问题分成独立的子问题，分别解决后合并结果
- 与 DP 的区别：子问题不重叠
- 经典题目：归并排序、快速排序、最近点对

**对比总结：**
| 算法 | 搜索空间 | 是否回退 | 结果 |
|------|---------|---------|------|
| 贪心 | 局部最优路径 | 不回退 | 近似/精确最优 |
| 回溯 | 所有可能解 | 会回退 | 所有解/最优解 |
| 分治 | 独立子问题 | 不回退 | 合并结果 |
| DP | 重叠子问题 | 不回退 | 全局最优 |

**得分点：**
- [ ] 清楚四种算法思想的核心区别
- [ ] 每种至少举一个经典题目
- [ ] 理解回溯的剪枝优化
- [ ] 能区分贪心和 DP 的适用条件

**加分项：**
- 分析贪心失败的反例
- 回溯问题中去重复解的技巧
- 能写出回溯算法的通用模板

**追问方向：**
- 如何判断一个题目该用贪心还是 DP？
- 回溯法的时间复杂度如何分析？
- 分治法在并行计算中有什么优势？

## Q5: 复杂度分析方法

**规范答案：**

**时间复杂度：**
- 表示算法执行时间随输入规模增长的变化趋势
- 常见级别（从低到高）：O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(2ⁿ) < O(n!)

**分析方法：**
1. **循环法**：统计循环次数
2. **递归法**：列递归方程，用主定理或递归树分析
3. **均摊分析**：如动态数组 push 的均摊 O(1)

**主定理（Master Theorem）：**
对于 T(n) = aT(n/b) + O(n^d)：
- a < b^d → O(n^d)
- a = b^d → O(n^d log n)
- a > b^d → O(n^(log_b(a)))

**空间复杂度：**
- 关注算法额外使用的内存空间
- 递归算法：栈深度也算空间复杂度
- 原地算法：O(1) 额外空间

**得分点：**
- [ ] 能正确分析循环和递归的时间复杂度
- [ ] 知道常见复杂度级别的大小关系
- [ ] 理解最好、最坏、平均情况的区别
- [ ] 了解空间复杂度的计算方法

**加分项：**
- 能用主定理分析分治算法的复杂度
- 理解均摊分析（Amortized Analysis）
- 能分析实际代码的复杂度（而非只知道理论）

**追问方向：**
- 快排的平均和最坏复杂度分别是多少？为什么？
- 哈希表的平均和最坏查找复杂度？
- O(n log n) 是比较排序的理论下界，为什么？

## Q6: 字符串算法

**规范答案：**

**KMP 算法：**
- 解决字符串匹配问题，时间复杂度 O(n+m)
- 核心：构建 `next` 数组（部分匹配表），失配时不回退主串指针
- `next[i]` 表示 pattern[0..i] 的最长真前缀等于真后缀的长度

**其他重要算法：**
| 算法 | 用途 | 时间复杂度 |
|------|------|-----------|
| Rabin-Karp | 字符串匹配（哈希滚动） | 平均 O(n+m) |
| Trie（前缀树） | 前缀匹配、自动补全 | 查询 O(L) |
| Manacher | 最长回文子串 | O(n) |

**常见字符串面试题：**
- 最长回文子串：中心扩展法 O(n²) 或 Manacher O(n)
- 字符串转整数（atoi）：注意溢出和边界
- 最长无重复字符子串：滑动窗口 O(n)

**得分点：**
- [ ] 理解 KMP 的核心思想（避免回退主串）
- [ ] 知道 next 数组的含义
- [ ] 能用滑动窗口解决子串问题
- [ ] 了解 Trie 的结构和应用

**加分项：**
- 能手写 KMP 算法
- 理解 Trie 在搜索引擎和输入法中的应用
- 提到后缀数组和后缀树

**追问方向：**
- 如何优化 KMP 的 next 数组（nextval）？
- Trie 和 HashMap 在字符串查找中各有什么优劣？
- 正则表达式的匹配算法了解吗？
